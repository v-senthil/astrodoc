1. Contact IM Team with the Use-case.

2. ZohoIM can be used in two Perspectives

3. Native Integration for Customers. Each customer can configure their own Number.

4. To Send Product Notifications(Similar to Trans-mail, Zoho Classes)

5. Provide your:

6. Service Type as in com.zoho.accounts.AccountsConstants.ServiceType

7. Org Type as in com.zoho.accounts.AccountsConstants.OrgType

8. Service Name to be configured in scoped-services

9. Store ID of your service

10. IM team will add a configuration in our end with your service details so that you can start using IM.

11. After a basic testing in csez, we shall start the process of payment add-ons. IM Team will request an IM add-on for your service to the store team.

12. Real time notifications through kafka [link](/1.0.0/portal/zohocorp/team/zohoim/manual/sdk-reference/article/real-time-notifications)
